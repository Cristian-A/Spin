Declarations

Spin declarations are statements that define variables, constants and functions.

Variable and Constant Definitions

The variable definition begins with the keyword var followed an identifier and an eventual type annotation. The same is true for constant definitions but they begin with the con keyword. Both variable and constant definition need an inline assignment if the type is not specified so that the language can deduce it from the assignment expression. The type annotation can also be called tag. The grammar is the following:

Production Rule

var declaration → 'var',identifier,tag,('=',expression)?,';'
| 'var',identifier,'=',expression,';';
con declaration → 'con',identifier,tag?,'=',expression,';';

Function and Procedure Definitions

Functions in spin always return an expression, effectively leaving it on top of the stack. Procedures on the other hand, don't produce any output. A function begins withe the keyword func while a procedure with the keyword proc, they are both followed by an identifier, parameters wrapped in round parenthesis (...). The function also requires a type annotation (tag) for the return specification. Both are followed by statements wrapped in curly braces {...}. The grammar follows:

Production Rule

func declaration → 'func',identifier,'(',parameters?,')',tag,'{',statements*,'}';
proc declaration → 'proc',identifier,'(',parameters?,')','{',statements*,'}';
