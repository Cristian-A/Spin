Spin Syntax Reference

This part of the documentation describes the formal grammar of the Spin programming language. The grammar described here is intended to help you understand the language in more detail.

The Spin language is relatively small, because many common types, functions, and operators that appear virtually everywhere in Spin code are actually defined in the Spin system standard. Although these types, functions, and operators are not part of the Spin language itself, they are used extensively in the discussions and code examples of the documentation.

How to read the Syntax Grammar

The notation used to describe the formal grammar of the Spin programming language follows a few EBNF conventions:

Production Rule

while statement → 'while','(',expression,')',statement;
  • The left-hand side of a production rule is the name of the rule.
  • An arrow (→) is used to mark grammar productions and separates the left-hand side from the right-hand side of a production rule.
  • Terminals and non terminals are separated by comma ,.
  • Terminals are always surrounded by single quotes '.
  • Non terminals are highlighted in blue while quantifiers in grey.
  • Terminals use constant width text and appear only on the right-hand side of a grammar production rule.
  • Alternative grammar productions are separated by vertical bars |.
  • When alternative productions are too long to read easily, they are broken into multiple grammar production rules on new lines.
  • Optional syntactic categories and literals are marked by a trailing quantifier (? optional, * zero or more, + one or more).
  • Syntactic categories can be grouped by parenthesis ().
  • The grammar production rule aways ends with a semicolon ;.

We also provide the RegExp equivalent of lexical rules and eventually, EBNF syntax will be followed by a RailRoad diagram of the syntax. Code can be shown to clarify certain concepts, however, the blue pills contained in the code are placeholders and can never be considered part of the language. Quotes, tables, notes and warnings of potential mistakes are scattered in form of small boxes throughout the reference.
